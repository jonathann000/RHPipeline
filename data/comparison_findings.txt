KEY FINDINGS
------------
1. BERT model choice matters significantly for text quality:
   - MeganTrainer: clean token boundaries, few false positives, but misses some names
   - Roberta: good recall, but fragments tokens ("Kung[ADRESS]", "älv", "ä")
   - OAI: highest recall, but extreme false positives and destroys text readability

2. LLM output is non-deterministic:
   - Roberta's first run: LLM found 0 entities
   - Roberta's second run: LLM found 5 entities
   - Same input, same model — confirms sampling variance in 8B model

3. All models consistently miss:
   - "Erik" in body text (coreference — name found in header but not linked to later mentions)
   - "Helena Björk" (doctor name — only OAI catches this, but with mangled output)
   - "Sahlgrenska" (hospital name)
   - "Huntingtons sjukdom" in Bedömning section (only caught in Aktuellt where it first appears)

4. OAI has a subword tokenization problem:
   - Redacts partial words: "S[ADRESS]enska", "[ADRESS]vs sjukhus", "Kung[ADRESS]älvs"
   - Redacts non-entities: "Återbesök" as person, "12" (years) as address, "hans" as person
   - Makes output unusable for research

RECOMMENDATIONS
---------------
1. Best model for research output: MeganTrainer
   - Most readable redacted text
   - Best LLM generalizations (preserves meaning while reducing identifiability)
   - Fewest false positives

2. Best model for privacy: Roberta (with LLM working)
   - Best name detection (catches first + last names)
   - "Kungälvs kommun" → "kommun i Västra Götaland" is excellent generalization
   - But token fragmentation hurts readability

3. Pipeline improvements needed (regardless of BERT model):
   - Coreference resolution: if "Erik Andersson" is found, redact all "Erik" mentions
   - Doctor name handling: add rule or prompt for "Dr." prefix detection
   - Hospital names: add to rule agent or entity list
   - Repeated entity check: "Huntingtons sjukdom" should be caught on every occurrence
   - Larger LLM (70B+) would improve consistency and reduce non-deterministic misses

4. OAI is not recommended in current form — needs post-processing to merge
   subword tokens into full-word entities before redaction.

REFLECTIONS
-----------
- The three-stage pipeline (rules → BERT → LLM) works well in principle.
  Rules handle structured data reliably, BERT catches named entities, and
  the LLM adds contextual understanding for quasi-identifiers.

- The LLM's ability to generalize (not just redact) is the most valuable
  feature for research — it preserves clinical meaning while reducing
  re-identification risk. A blanket [REDACTED] loses information;
  "hantverkare" instead of "skorstensfejare" keeps the occupational context.

- The 8B LLM is borderline for this task. It works but is inconsistent.
  A larger model would likely be more reliable at identifying quasi-identifiers
  and producing consistent generalizations across runs.

- The biggest gap is not the models themselves but the pipeline logic:
  coreference resolution and entity propagation would fix the most critical
  leaks (repeated names) without changing any model.
